# FlowVium 프로젝트 규칙

## 🔒 FEATURES.md 유지 의무 (필수)

이 저장소에 UI 기능 또는 백엔드 엔드포인트를 **추가·수정·삭제**할 때마다 **반드시** 루트의 `FEATURES.md` 파일을 같은 작업에서 업데이트한다.

### 적용 범위 (트리거 목록)

| 변경 사항 | FEATURES.md 반영 위치 |
|----------|----------------------|
| 새 페이지 추가 | 새 섹션 생성 (목차 포함) |
| 새 탭 추가 | 해당 페이지 섹션에 탭 서브섹션 추가 |
| 새 카드·위젯·모달 추가 | 해당 탭 불릿 추가 |
| 기존 기능 제거 | 해당 불릿/섹션 삭제 |
| 🔒 잠금 해제 | 🔒 배지 제거 + 데이터 소스 갱신 |
| 데이터 소스 교체 | API 라우트 테이블 데이터 소스 컬럼 수정 |
| 새 API 라우트 | "18. 백엔드·인프라 기능 → API 라우트" 테이블에 행 추가 |
| API 라우트 삭제 | 해당 행 삭제 |
| 새 크론 잡 | "자동화 크론" 테이블에 행 추가 |
| 캐시 TTL 변경 | 해당 행 TTL 컬럼 수정 |
| 새 공유 컴포넌트 | "17. 공유 UI 컴포넌트" 테이블에 행 추가 |

### 워크플로
1. 코드 수정
2. `FEATURES.md` 해당 섹션 **즉시** 수정
3. 코드 변경과 **같은 작업**에서 처리 (별도 작업으로 미루기 금지)

### 누락 시
- 다음 세션 시작 시 감사 후 역추적 수정
- FEATURES.md가 코드와 불일치하면 항상 코드를 기준으로 FEATURES.md를 맞춤

## 🗂️ 기타 프로젝트 관습

- i18n: 모든 UI 문자열은 `messages/*.json`에 넣고 하드코딩 금지
- 16개 언어 동시 업데이트 (ko/en/ja/zh-CN/zh-TW/es/fr/de/pt/ru/ar/hi/id/th/tr/vi)
- Redis 쓰기: 직접 `redis.set` 금지, `loggedRedisSet` 사용
- 외부 fetch: 가능하면 `loggedFetch` 사용 (자동 REDACT + 타이밍)
- 유료 API 탭 락 메시지: "사용자에게 API 키 입력 요구 금지" — "월 $200 후원 목표 도달 시 오픈" 형식만 사용
- `research_history/YYYY-MM-DD_*.txt`에 모든 작업 마일스톤 기록
